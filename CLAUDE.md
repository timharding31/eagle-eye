# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this app is

Eagle Eye is a personal Android golf rangefinder, sideloaded as an APK. Expo + React Native, fully offline at the course. Solo-dev project — no test suite. Architecture (deep modules with narrow interfaces) is the primary debugging aid; bugs are meant to localize to a single module via its small interface.

Read these two files before doing any non-trivial work — they are load-bearing:

- `CONTEXT.md` — the domain glossary. Use these terms **exactly** in code and docs (Course, Hole, Green, Pin, Round, Tee, Landing Zone, etc.). The "Terms to avoid" list there is enforced (no "yardage", "tracker", "user", "API"). Add a term to this file before naming a new domain concept in code.
- `docs/PLANNING.md` — the phasing plan, tech-stack rationale, module table, data model, and UX flows. The module table is the source of truth for which file owns which capability.
- `docs/adr/` — ten ADRs that lock in non-obvious decisions (front/back via closest point, SQLite as source of truth, source-agnostic course data, manual hole nav, MapLibre offline packs, tee-box deferred, Find Nearby only, prefetch both tile layers, tee-correction overlay, shot-recording dropped). Check these before changing related behavior.
- `docs/UI_LAYOUT.md` — maps the hole-view pixels to the files in `components/hole/` that own them; read it before touching the play screen's layout.

## Commands

```bash
npm start                                    # Expo dev server (custom dev client, NOT Expo Go)
npm run android                              # Launch on connected Android device/emulator
npm run lint                                 # Expo's ESLint config + prettier
npm run format                               # Prettier write
npm run format:check                         # Prettier check
npm run db:generate                          # Drizzle: regenerate migrations from lib/**/schema.ts
npm run build:course -- <type/id> <slug>     # Node script: Overpass → courses/<slug>.json
                                             #   e.g. npm run build:course -- way/16650363 presidio
```

There are no tests. Do not add a test runner unless asked — the architecture choice is to skip them and rely on deep-module interface isolation (see PLANNING.md "How architecture eases debugging without tests").

`npm install` uses `legacy-peer-deps=true` (see `.npmrc`) — needed for the Expo 56 / RN 0.85 / React 19 dep graph. Don't fight it.

## Architecture: deep modules in `lib/`

The repo is grouped by domain concept, not by layer. Each `lib/<module>/index.ts` is the **only** way other code should talk to that module — internal seams (Drizzle, Turf, MapLibre, Zustand, GPS) must not leak through the interface.

| Module       | Owns                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      | Internal seams                                                      |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| `lib/geo`    | Pure geospatial math (`distanceMeters`, `nearestPointOnPolygon`, `farthestPointOnPolygon`, `centroid`, `pointInPolygon`, `bearingDeg`, `frameForHole`, `lzInitPositions`, `projectionFraction`, `bboxOf`).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                | Turf.js — wrapped, never re-exported.                               |
| `lib/course` | Course loading + normalization + the Overpass adapter (Find Nearby discovery, fetch, tap-to-fix green synthesis, SQLite install) + per-course Tee Corrections (`setTeeOverride`, applied as an overlay in `loadCourse` — see ADR-009). Source-agnostic (`bundled` / `overpass` / `ml` all produce same `Course` shape). `normalize.ts` is shared with the Node build script.                                                                                                                                                                                                                                                                                                              | OSM tag parsing, Drizzle, Overpass HTTP, Zustand (pending-install). |
| `lib/round`  | Round lifecycle + the **single-active-round invariant** (at most one row in `rounds` with `ended_at IS NULL`). Owns the Zustand store + SQLite hydration.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 | Drizzle, Zustand, stale-round (>24h) check.                         |
| `lib/tiles`  | Offline tile management (raster + vector) per Course. Vector packs go to z16–20; satellite to z16–20, with **Presidio** uniquely to z21 (`SATELLITE_MAX_ZOOM_BY_COURSE`). Exposes `vectorStyle`, `satelliteStyleFor(courseId)`, `satelliteMaxZoom(courseId)`, `prefetchForCourse`/`Status`/`retryPrefetch`. The on-screen render style oversamples (`SATELLITE_RENDER_TILE_SIZE = 128`) so the overview pulls sharper tiles than the camera zoom alone would. The offline downloader can only fetch the style over http(s), so the satellite style is **hosted on GitHub Pages** (`docs/satellite-style.json`), not inlined — keep its `maxzoom` in sync with `SATELLITE_SOURCE_MAXZOOM`. | MapLibre offline manager, URL templates, ESRI World Imagery.        |

Screens in `app/` (Expo Router, file-based) stay thin — if one grows past ~200 lines doing math or parsing inline, push it into a module. Path alias `@/*` resolves to the repo root (see `tsconfig.json`). The two screens with real complexity follow the same shape: a thin route loader in `app/` over a `components/<screen>/` module. The hole view is the reference (below); the home screen mirrors it in **`components/home/`** — `app/index.tsx` only does hydration + the permission gate, then renders `HomeSceneProvider` → `HomeLayout`, with `CourseList` and `ActiveRoundCard` as regions consuming `useHomeScene()`. The flatter screens (history, courses/add, courses/fix, round/scorecard) stay single-file.

### Hole view (the play screen) — `components/hole/`

`app/round/[hole].tsx` is now just a route loader: it resolves the active round + its course, guards loading/not-found, and hands the resolved scene to a provider. Everything else lives under `components/hole/`:

- **`scene.tsx`** — `HoleSceneProvider` + `useHoleScene()`. The single owner of GPS (the `watchPositionAsync` subscription), derived geometry (`teeLL`, `greenC`, `pin`, `holeYards`), cross-region toggles (`cameraMode`, `lzShown`), hole navigation, and the tee-correction lifecycle. Rule: state **more than one region** needs lives here; state only **one region** needs stays local to that region (F/G/B distance math is in `HoleMeasurements`; LZ positions + camera framing are in `HoleMap`).
- **`HoleLayout.tsx`** — pure composition; stacks the regions over the map under `GlassRoot`.
- Regions: `HoleMap`, `HoleMeasurements` (F/G/B pill + Distance from Tee pill), `HoleButtonStack` (LZ / Green-Hole / Set Tee), `BottomDrawer` (prev/next + hole-grid picker), `HoleHeader`, `TeeOverrideDialog`. `units.ts` holds `M_TO_YD`.
- **Hole navigation uses `router.setParams`, not `replace`** — a replace tears down the screen and remounts a fresh `HoleMap`, snapping the camera. setParams keeps the live map mounted so the camera eases to the next hole.
- **Tee-shot recording was removed** (ADR-010): no `lib/shots`, no `tee_shots` table. The hole screen instead shows a passive **Distance from Tee** readout, made trustworthy by the Set Tee correction (ADR-009).

## Data layer specifics

- **SQLite is the source of truth** for rounds + course install state (ADR-002). Active-round resume on cold launch goes through `lib/round.ensureHydrated()`, called from `app/_layout.tsx` after `useMigrations` succeeds.
- **Drizzle schemas** live next to the module that owns them (`lib/course/schema.ts`, `lib/round/schema.ts`). `drizzle.config.ts` globs `./lib/**/schema.ts`. `db/index.ts` is the single drizzle client.
- **Migrations**: the runtime applies a hand-maintained static bundle at `db/migrations.ts` (imported in `app/_layout.tsx`, fed to `useMigrations`) — the gitignored `drizzle/` inline-import path was replaced by this bundle so cloud EAS builds don't depend on `.sql` import resolution. After a schema change, add a new `mNNNN` migration string + a matching journal entry (next `idx`, later `when`) to `db/migrations.ts`; don't edit an already-shipped migration (it won't re-run on installed devices). `npm run db:generate` still regenerates the `drizzle/` snapshot for dev reference, but it is not what runs at startup.
- **Course JSON** in `courses/` is committed and bundled into the APK (`presidio`, `harding-park`, `crystal-springs`, `lincoln-park`, `peacock-gap`). Each is wired into `BUNDLED_REGISTRY` in `lib/course/index.ts`, keyed by slug; a new bundled course needs both the JSON file and a registry entry there. (`presidio` is the home course — OSM way 16650363.) Courses added at runtime via Find Nearby live in SQLite, not in `courses/`.

## Conventions worth knowing

- TypeScript strict mode is on. Prefer types over `any` — the geo code is null-and-edge-case heavy and types catch real bugs.
- Prettier: no semicolons, single quotes, trailing commas, `arrowParens: avoid`.
- The `Course` type uses GeoJSON-style `Position = [lng, lat]` ordering (see `lib/course/types.ts`). Internal app code uses `LatLng = { lat, lng }`. Conversions happen at the `lib/geo` boundary — don't sprinkle `[1, 0]` swaps in screens.
- **UI is an in-house design system in `lib/theme.ts`** (`colors`, `space`, `radius`, `fonts`, `type`, `shadows` — deep-navy surfaces, cream text, maroon CTA, Sora typeface loaded in `_layout.tsx`). The palette is **generated from a handful of oklch "knob" bases** at the top of `theme.ts` (`_surface`, `_primary`, `_cta`, …) via a small oklch→sRGB pipeline; every derived color is a `shade()`/alpha variant of a knob. **Retheme by editing the knobs, not the derived `colors`.** Screens compose plain RN `StyleSheet` with the leaf primitives in `components/` (`Button`, `SectionLabel`, `EagleIcon`, `GlassSurface`, `GlassHeader`, `MapBackdrop`) and SVG glyphs in `components/icons.tsx` (hand-rolled paths + a few re-exports from `lucide-react-native`). Style off `@/lib/theme`, not hardcoded hex. There is no third-party UI kit — Gluestack was removed. **Every screen now wears the same glass-over-satellite chrome**: float `GlassSurface` panels over a `MapBackdrop` (the still-satellite home/list backdrop) or a live map, with `GlassHeader` as the shared top bar (home shape = logo + wordmark + menu; sub-screen shape = back + title + subtitle). The old solid `Card`/`ScreenShell`/`TopBar` primitives were removed.
- **Glass chrome over the map uses real backdrop blur** (`components/GlassSurface.tsx`, `expo-blur`), not opaque cards. Wrap the screen in `GlassRoot`, the map in `GlassBlurTarget`, and float `GlassSurface`/`GlassBackdrop` panels on top; the blur target is threaded via context. Android's SDK-31+ dimezis blur does **not** auto-sample the hierarchy — it needs the `blurTarget` ref, and it can only capture a `texture`-backed map (see the next bullet). The navy tint (`glassFill`/`glassFillDark`) is a separate overlay _above_ the BlurView, since BlurView paints over its own background. Keep glass panels small/few — they re-blur every frame.
- Distances are computed in metres throughout `lib/geo`; the hole screen converts to **yards** (golf default) only at the display boundary (`M_TO_YD` in `components/hole/units.ts`). **Yards-only is intentional — a units toggle is a won't-do**, and there's no settings store or in-app settings panel (deferred; see PLANNING.md Phase 7). Still don't bake "yards" into names or stored values — the metres-internal / yards-at-the-boundary split stays.
- Map renderer is `@maplibre/maplibre-react-native` — requires the custom dev client, won't run in Expo Go. The hole map mounts with `androidView="texture"` (not the default `"surface"`/GLSurfaceView) so the glass blur can capture it. On-map text uses `Marker` (native RN view), not symbol/text layers — the satellite raster style has no glyphs URL and text-field layers crash on it.
